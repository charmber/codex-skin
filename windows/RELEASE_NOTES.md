# Codex Dream Skin 1.11.5

## Windows

- 经典蓝布局的企鹅账户卡片只在任务侧栏显示，不再覆盖设置分类或把“编码”分组压成 72px
- Windows 设置侧栏和正文取消 `-46px` 上移，从原生菜单下方开始布局，较矮窗口交给 Codex 自带滚动区处理
- 主题工作室“颜色”页新增“聊天记录文字”，可单独调整用户提问和 Codex 回答的正文颜色；旧主题默认继续沿用“主文字”颜色
- 经典蓝 QQ 工作台在 Windows 上不再把右上角原生状态按钮复制到复古标题栏，避免与 Codex 自带状态图标重叠
- 收紧左上角 `Codex / Work` 模式切换按钮的宽度与层级，减少与相邻按钮挤压后出现的遮挡和点不到问题
- Windows 任务页不再把侧栏首行抬进原生菜单栏，彻底分离 `Codex / Work` 与隐藏边栏、返回、前进按钮的点击区域
- 设置页保留新版分类导航的包装层，修复进入设置后左侧分类整栏消失的问题
- 为经典蓝复古标题栏补上 Windows 可拖拽区域，同时把工具栏和可点击控件排除出拖拽命中区，恢复窗口拖动能力
- 提供安装版与便携版 x64 EXE，以及 `SHA256SUMS-Windows.txt`

## macOS

- 经典蓝布局进入设置时不再把任务侧栏的账户卡片注入分类列表，避免卡片绝对定位覆盖设置项并压缩分组高度
- 主题渲染器支持可选的聊天记录文字颜色；从 Windows 主题工作室导入并应用的主题会在未来舞台和经典蓝布局中显示该配色，旧主题继续沿用主文字颜色
- 设置页保留新版 Codex 的分类导航包装层，避免界面结构更新后左侧分类被主题样式隐藏

## 安全边界

- Windows 与 macOS 都只通过 `127.0.0.1` 回环 CDP 注入
- 不修改官方应用、`app.asar`、代码签名、账户或 API 配置
- Windows 主题包仍只允许声明式 JSON、图片和说明/许可文件，不执行包内代码

Windows EXE 当前未签名，SmartScreen 可能显示“未知发布者”。请只从本仓库 Release 下载并核对 SHA-256。
